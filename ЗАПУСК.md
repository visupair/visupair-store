# Инструкция по запуску — Делайте вручную

## ⚠️ Проблема с запуском dev-сервера

Есть проблема с правами доступа к папке `.config/.wrangler` в WSL.

---

## 🛠️ Как исправить (вручную):

### Вариант 1: Удалите папку .wrangler

1. Откройте терминал в VS Code (или WSL терминал)
2. Выполните:

```bash
cd ~
rm -rf .config/.wrangler
mkdir -p .config/.wrangler/logs
mkdir -p .config/.wrangler/registry
```

3. Запустите dev-сервер:

```bash
cd ~/visupair-store
npm run dev
```

### Вариант 2: Если не помогло — запустите с sudo

```bash
cd ~/visupair-store
sudo npm run dev
```

(Потребуется ввести пароль)

### Вариант 3: Отключите platformProxy временно

Если всё равно не работает, можно временно отключить Cloudflare platformProxy для локальной разработки:

1. Откройте файл `astro.config.mjs`
2. Измените:

```javascript
adapter: cloudflare({
  platformProxy: {
    enabled: false, // ← Измените на false
  },
  imageService: 'cloudflare',
}),
```

3. Сохраните и запустите:

```bash
npm run dev
```

**Внимание:** При `enabled: false` локально не будут работать:
- Cloudflare KV (сессии)
- Better Auth авторизация

Но статические страницы (/, /about) будут работать нормально!

**Что должно работать:**

1. Откройте Chrome DevTools (F12)
2. Нажмите "Toggle Device Toolbar" (Ctrl+Shift+M)
3. Выберите iPhone/Android
4. Перезагрузите страницу (Ctrl+Shift+R)
5. В Network tab выберите "Fast 3G" или "Slow 3G"
6. Снова перезагрузите

**Что должно произойти:**
- ✅ Изображение появляется **сразу** (даже на медленном интернете)
- ✅ Анимация загружается в фоне
- ✅ Никаких пустых экранов

### 3. Проверьте вход/выход:

1. Зарегистрируйтесь или войдите (если авторизация работает локально)
2. Нажмите кнопку "Account" (справа вверху)
3. Должна появиться боковая панель с вашим именем
4. Нажмите "Sign out"

**Если авторизация не работает локально** (из-за platformProxy: false):
